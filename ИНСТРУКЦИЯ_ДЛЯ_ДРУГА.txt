ПЕРЕДАЧА ПРОЕКТА ДРУГУ (АКТУАЛЬНАЯ ВЕРСИЯ)
=========================================

1) Какой файл самый важный и "свежий" по смыслу
-----------------------------------------------
Главный файл проекта: app.py

Почему:
- В конфигурации проекта (README front matter) указан запуск через app.py
- В app.py находится основная логика: детекция сонливости, загрузка ML-моделей, работа с картой/остановками
- По истории git это основной рабочий файл приложения

Важно:
- src/streamlit_app.py сейчас выглядит как стандартный demo-шаблон Streamlit (спираль), а не основной прод-код.


2) Что обязательно отправить другу (минимальный рабочий набор)
--------------------------------------------------------------
Обязательно:
- app.py
- requirements.txt
- bus_stops.json
- eye_classifier.pkl
- eye_scaler.pkl
- class_mapping.pkl

Желательно тоже отправить:
- README.md
- images/ (если используются картинки/иконки)

Если хочешь, чтобы друг точно запустил без сюрпризов:
- Проще всего передавать весь проект целиком (zip папки), а не 1 файл.


3) Что это за "модель" в твоем проекте
--------------------------------------
В проекте используется обученная ML-модель для классификации состояния глаз:
- eye_classifier.pkl  -> сам классификатор
- eye_scaler.pkl      -> нормализация признаков перед классификацией
- class_mapping.pkl   -> соответствие числового класса к метке (например, "спит/не спит")

В app.py модель загружается через joblib.load(...) в функции load_model().


4) Быстрый запуск для друга (Windows / PowerShell)
--------------------------------------------------
Шаги:
1. Открыть папку проекта в терминале
2. (Рекомендуется) создать виртуальное окружение:
   python -m venv .venv
   .\.venv\Scripts\activate
3. Установить зависимости:
   pip install -r requirements.txt
4. Запустить приложение:
   streamlit run app.py


5) Что написать другу одним сообщением (готовый текст)
------------------------------------------------------
"Основной код в app.py. Это рабочая версия.
Чтобы запустить: pip install -r requirements.txt и streamlit run app.py.
Не забудь, что для работы модели нужны файлы eye_classifier.pkl, eye_scaler.pkl, class_mapping.pkl и bus_stops.json."


6) Рекомендация по передаче "самого нового кода"
------------------------------------------------
Если нужен именно "самый новый код", лучше передавать:
- либо всю папку проекта zip-архивом;
- либо ссылку на git-репозиторий/ветку с последним коммитом.

Передача только одного app.py может не запуститься у друга без .pkl и остальных файлов.
